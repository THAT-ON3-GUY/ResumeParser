## 1. Problem Statement

Beyond LinkedIn, candidates leave traces in free public databases — professional license registries, court records, SEC filings, business filings, and GitHub. These sources can corroborate resume claims (e.g., verify a PE license is real) or surface information not on the resume (e.g., a court record or business filing). All sources used in this feature are free public APIs.

## 2. Goals

- Query relevant free public APIs based on what was extracted from the resume
- Only query sources relevant to the candidate's apparent field
- Return structured results for each source checked
- Run all queries in parallel to minimize total search time

## 3. Non-Goals

- Paid data sources (no LexisNexis, no Spokeo, no BeenVerified)
- Social media beyond GitHub (no Twitter/X, no Facebook)
- Sources requiring authentication beyond a free API key
- Any source that would constitute a consumer report under FCRA

## 4. Functional Requirements

- **FR-1.** The system MUST query the NPI Registry if `licenses_certifications` contains any healthcare-related term (e.g., MD, RN, NP, PA, DO, DDS, PharmD).
- **FR-2.** The system MUST query FINRA BrokerCheck if `licenses_certifications` contains finance-related terms (e.g., Series 7, Series 63, CFP, CFA).
- **FR-3.** The system MUST query OpenCorporates officer search if any employer name was extracted.
- **FR-4.** The system MUST query GitHub user search if `skills` contains software-related terms (e.g., Python, JavaScript, React, SQL, AWS).
- **FR-5.** The system MUST query CourtListener if the candidate's field suggests it is relevant (e.g., law, legal).
- **FR-6.** All queries MUST run in parallel using `Promise.allSettled()` — a failure in one source MUST NOT block others.
- **FR-7.** The system MUST return results within 15 seconds total — any source that does not respond in time is skipped.
- **FR-8.** The system MUST store which sources were checked and what was found, including empty results.

## 5. Acceptance Criteria

- **AC-1.** *Given* a resume with "PE License" in certifications, *when* public records run, *then* NPI registry is NOT queried (PE is engineering, not healthcare) and the correct engineering license board is noted as a manual check suggestion.
- **AC-2.** *Given* a resume with "Series 7" in certifications, *when* public records run, *then* FINRA BrokerCheck is queried and results are stored.
- **AC-3.** *Given* a resume with Python and React in skills, *when* public records run, *then* GitHub is queried for matching users.
- **AC-4.** *Given* one public API times out, *when* results are collected, *then* that source shows "Timed out" and all other sources complete normally.

## 6. Data Model

```typescript
{
  sourcesChecked: string[],
  results: {
    npi?: { found: boolean, data: object | null },
    finra?: { found: boolean, data: object | null },
    openCorporates?: { found: boolean, data: object | null },
    github?: { found: boolean, profiles: object[] },
    courtListener?: { found: boolean, data: object | null }
  },
  checkedAt: string
}
```

Stored in `public_records` JSON column of candidates table.

## 7. API Surface

**File:** `src/main/search/publicSources.js`

```typescript
checkPublicSources(extractedFields: ExtractionSchema): Promise<PublicRecordsResult>
```

APIs called:
- `https://npiregistry.cms.hhs.gov/api/` — no key required
- `https://api.brokercheck.finra.org/search/individual` — no key required
- `https://api.opencorporates.com/v0.4/officers/search` — no key required (rate limited)
- `https://api.github.com/search/users` — no key required (60 req/hr unauthenticated)
- `https://www.courtlistener.com/api/rest/v3/people/` — no key required

## 8. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| GitHub unauthenticated rate limit (60/hr) | M | M | Cache results per candidate; add GitHub token field to Settings (optional) |
| OpenCorporates free tier rate limit | M | L | Results are supplementary; skip gracefully if rate limited |
| Wrong person matched in public records | H | M | Always show raw results and let recruiter judge; never auto-confirm a match |
