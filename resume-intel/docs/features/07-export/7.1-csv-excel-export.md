## 1. Problem Statement

Recruiters often need to share candidate findings with hiring managers or log them in an ATS. The app needs to export results to formats those workflows already accept — CSV and Excel.

## 2. Goals

- Export all results or a selected subset to CSV or Excel
- Excel export includes formatting and a separate Sources sheet
- Export is triggered from the UI with a single click
- Files save to the user's Downloads folder by default

## 3. Functional Requirements

- **FR-1.** The `ExportBar` component MUST offer two export options: "Export CSV" and "Export Excel".
- **FR-2.** CSV export MUST produce one row per candidate with one column per extracted top-level field.
- **FR-3.** Excel export MUST produce a formatted sheet with color-coded confidence column (green/amber/red).
- **FR-4.** Excel export MUST include a second sheet named "Sources" listing all URLs found per candidate.
- **FR-5.** The system MUST allow exporting all candidates or only selected rows (checkbox selection in table).
- **FR-6.** Export MUST save to the OS Downloads folder and open the file after saving using `shell.openPath()`.
- **FR-7.** The system MUST use the `xlsx` library in the main process — never in the renderer.

## 4. Acceptance Criteria

- **AC-1.** *Given* 5 parsed candidates, *when* "Export Excel" is clicked, *then* an .xlsx file opens with 5 data rows, formatted headers, and a Sources sheet.
- **AC-2.** *Given* 2 rows are selected in the table, *when* "Export CSV" is clicked, *then* only those 2 rows appear in the CSV.
- **AC-3.** *Given* export completes, *when* the file is saved, *then* it opens automatically in Excel or the default spreadsheet app.

## 5. UI / UX

`ExportBar` at the top of the results table:
- "Export All" dropdown: CSV | Excel
- "Export Selected (N)" button — only visible when rows are checked
- Progress spinner during export
- Success toast: "Exported to Downloads/resume-intel-export-2026-06-23.xlsx"

## 6. Data Model

No new DB tables. Reads from `candidates` table via `db.getAllCandidates()` or `db.getCandidatesByIds(ids)`.
