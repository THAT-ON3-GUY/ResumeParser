## 1. Problem Statement

LinkedIn profile scraping requires an authenticated session. The recruiter must sign in once through a visible browser window; the app saves session cookies locally and reports connection status in Settings.

## 2. Goals

- Open a `BrowserWindow` to `https://www.linkedin.com/login`
- Detect successful login (feed/home URL navigation)
- Persist `.linkedin.com` cookies via `electron-store`
- Expose Connect / Disconnect in Settings with 🟢/🔴 status
- Notify renderer via `linkedin-connected` IPC event

## 3. Functional Requirements

- **FR-1.** Settings MUST show LinkedIn Connected / Not Connected.
- **FR-2.** Disconnect MUST clear stored cookies and set `linkedinConnected: false`.
- **FR-3.** Search pipeline MUST prompt connect when session missing (no silent failure).

## 4. Acceptance Criteria

- [ ] User connects LinkedIn once; status persists across app restarts
- [ ] Disconnect forces re-login before next profile scrape
- [ ] No LinkedIn credentials stored — only session cookies
