## 1. Problem Statement

The app has multiple configurable values — API keys, AI provider choice, LinkedIn session, search behavior — that need a stable, accessible home. Without a Settings panel, users have no way to configure or reconfigure the app after initial setup, and API keys are hardcoded or set via environment variables which a non-technical user cannot manage.

## 2. Goals

- Provide a single Settings panel accessible from the sidebar
- Allow the user to enter and update all API keys
- Allow switching between AI providers
- Show LinkedIn connection status and allow reconnect/disconnect
- Persist all settings to electron-store

## 3. Functional Requirements

- **FR-1.** Settings MUST be accessible via a gear icon in the sidebar — no menu bar navigation.
- **FR-2.** The following fields MUST be present: AI Provider toggle (Gemini / Claude), Gemini API Key input, Claude API Key input, Google Custom Search API Key (optional), Google CX ID (optional), LinkedIn Connect/Disconnect, Search Delay slider (2-10 seconds), Auto-search on upload toggle, Default export format (CSV / Excel).
- **FR-3.** All values MUST be saved to `electron-store` on change (not requiring a Save button).
- **FR-4.** API key fields MUST be password-type inputs (characters hidden) with a show/hide toggle.
- **FR-5.** Each API key field MUST show a status indicator: 🟢 Set / 🔴 Not Set.
- **FR-6.** The Gemini key field MUST include a helper link: "Get a free key at aistudio.google.com →".
- **FR-7.** The Claude key field MUST include a helper link: "Get a key at console.anthropic.com →" and a cost warning: "~$0.01-0.02 per resume".
- **FR-8.** Clicking helper links MUST open URLs in the system browser via `shell.openExternal()`.
- **FR-9.** The Settings panel MUST include a "Clear All Data" button that deletes the SQLite database after a confirmation dialog.

## 4. Acceptance Criteria

- **AC-1.** *Given* the user enters a Gemini API key and closes Settings, *when* the app is restarted, *then* the key is still present and the status shows 🟢 Set.
- **AC-2.** *Given* Claude is selected as the AI provider, *when* a resume is uploaded, *then* Claude is used for extraction.
- **AC-3.** *Given* the user clicks "Get a free key at aistudio.google.com →", *when* clicked, *then* the URL opens in their default browser.
- **AC-4.** *Given* the user clicks "Clear All Data" and confirms, *when* confirmed, *then* the candidates table is emptied and the results table shows the empty state.

## 5. UI / UX

- Full-panel view replacing the main content area when Settings is open
- Sections: AI Provider, API Keys, LinkedIn, Search Behavior, Data Management
- Each section has a clear heading and subtle divider
- No Save button — all settings auto-save on change
- Success flash on each field when saved: "Saved ✓"

## 6. Data Model

All stored in electron-store (key-value):
```javascript
aiProvider: "gemini" | "claude"
geminiApiKey: string
claudeApiKey: string
googleSearchApiKey: string
googleCxId: string
linkedinCookies: Cookie[]
linkedinConnected: boolean
linkedinConnectedAt: string
searchDelaySeconds: number  // 2-10
autoSearchOnUpload: boolean
defaultExportFormat: "csv" | "excel"
```
