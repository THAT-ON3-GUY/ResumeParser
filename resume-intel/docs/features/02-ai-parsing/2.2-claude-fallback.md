## 1. Problem Statement

Gemini is the default free provider but recruiters may want higher extraction accuracy for difficult resumes. Claude Sonnet is the optional paid upgrade, selectable in Settings. All calls must still route through `aiProvider.js` — never direct API usage elsewhere.

## 2. Goals

- Implement `claudeParser.js` using shared prompts from `prompts.js`
- Wire `aiProvider.js` to route to Claude when `store.get('aiProvider') === 'claude'`
- Read API key from `electron-store` (`claudeApiKey`)
- Match Gemini error handling: strip fences, JSON parse with repair fallback

## 3. Functional Requirements

- **FR-1.** Settings toggle MUST switch between `gemini` and `claude` without code changes.
- **FR-2.** Claude model MUST default to `claude-sonnet-4-20250514` (configurable later).
- **FR-3.** Missing Claude key MUST surface a clear Settings error, not a silent fallback to Gemini.

## 4. Acceptance Criteria

- [ ] User selects Claude in Settings and parses a resume successfully
- [ ] `aiProvider` logs which provider was used per record
- [ ] No renderer or search module imports Anthropic SDK or calls Claude directly
