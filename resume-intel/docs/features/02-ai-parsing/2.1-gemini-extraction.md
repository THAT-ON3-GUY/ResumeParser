## 1. Problem Statement

Raw resume text is unstructured and inconsistently formatted. The app needs to convert it into a reliable JSON schema that downstream search and UI features can depend on. Gemini 1.5 Flash is the primary provider because it is free (1,500 req/day). Currently basic extraction works but the output is missing source quotes, confidence scores, and several fields added after Milestone 1.

## 2. Goals

- Extract all defined candidate fields from raw resume text
- Return structured JSON that matches the canonical schema exactly
- Never hallucinate — return null for missing fields, never guess
- Include a source quote for every non-null field for traceability
- Assign a per-field confidence score

## 3. Non-Goals

- Identifying the candidate by name (intentionally excluded)
- Comparing against previous resumes
- Any network calls — this feature only calls the Gemini API
- Claude fallback logic (covered in 2.2)

## 4. Personas & User Stories

- **As a recruiter**, I want every field from the resume captured accurately so that I can search for the candidate without missing key signals like a niche certification or unusual employer.
- **As a recruiter**, I want to see where each extracted fact came from in the original resume so that I can verify the AI didn't make something up.
- **As a developer maintaining this app**, I want a consistent JSON schema so that I never have to handle undefined fields in the UI or database.

## 5. Functional Requirements

- **FR-1.** The system MUST call the Gemini 1.5 Flash API using a raw `fetch()` call — no SDK.
- **FR-2.** The system MUST use the API key stored in `electron-store` under `geminiApiKey`.
- **FR-3.** The system MUST set `temperature: 0.1` and `maxOutputTokens: 4000` on every Gemini call.
- **FR-4.** The system MUST strip markdown code fences from the response before attempting JSON.parse().
- **FR-5.** The system MUST extract all fields defined in the canonical schema below.
- **FR-6.** The system MUST return `null` or `[]` for any field not found — never infer or estimate.
- **FR-7.** The system MUST include a `source_quote` string for every non-null extracted field.
- **FR-8.** The system MUST include a `confidence` value ("high" | "medium" | "low") per field.
- **FR-9.** The system MUST throw a descriptive error if the API key is missing.
- **FR-10.** The system MUST throw a descriptive error if JSON parsing fails, including the first 300 chars of the raw response for debugging.
- **FR-11.** The system SHOULD retry once with a 2-second delay if the Gemini API returns a 429 (rate limit) error.

## 6. Non-Functional Requirements

- **Performance** — API call MUST complete within 15 seconds. If it exceeds this, throw a timeout error.
- **Cost** — Each extraction call MUST use fewer than 4,000 output tokens. Monitor via response metadata.
- **Privacy** — Resume text is sent to Google's Gemini API. This must be disclosed in the Settings panel.
- **Reliability** — Failure on one resume MUST NOT affect other resumes in a batch.
- **Maintainability** — All prompt text lives exclusively in `prompts.js`. No prompt strings in `geminiParser.js`.

## 7. Acceptance Criteria

- **AC-1.** *Given* a valid resume with 5 employers, *when* extraction runs, *then* all 5 employers appear in `previous_employers` with company, title, start_year, and end_year.
- **AC-2.** *Given* a resume that does not mention pronouns, *when* extraction runs, *then* `pronouns` is `null` not an empty string or a guess.
- **AC-3.** *Given* Gemini wraps its response in ```json code fences, *when* the parser receives it, *then* the fences are stripped and JSON.parse() succeeds.
- **AC-4.** *Given* a missing Gemini API key, *when* extraction is called, *then* the error message reads "Gemini API key not set. Add it in Settings."
- **AC-5.** *Given* a resume with a PE license, *when* extraction runs, *then* `licenses_certifications` contains an entry with `name: "PE License"` and a `source_quote` from the resume text.
- **AC-6.** *Given* a 429 rate limit response, *when* the parser receives it, *then* it waits 2 seconds and retries once before throwing.

## 8. Data Model

**Canonical extraction schema** (output of this feature, input to DB and UI):

```typescript
{
  previous_employers: Array<{
    company: string,
    title: string,
    start_year: number | null,
    end_year: number | null,
    source_quote: string,
    confidence: "high" | "medium" | "low"
  }>,
  education: Array<{
    school: string,
    degree: string,
    field: string,
    graduation_year: number | null,
    source_quote: string,
    confidence: "high" | "medium" | "low"
  }>,
  licenses_certifications: Array<{
    name: string,
    issuing_body: string | null,
    year: number | null,
    source_quote: string,
    confidence: "high" | "medium" | "low"
  }>,
  skills: string[],
  pronouns: string | null,
  years_experience: number | null,
  location_hints: string[],
  associations_memberships: string[],
  languages: string[],
  contact_hints: string[],
  summary_title: string | null
}
```

## 9. API Surface

**File:** `src/main/parser/geminiParser.js`

Exported functions:
```typescript
extractWithGemini(resumeText: string): Promise<ExtractionSchema>
summarizeWithGemini(data: object): Promise<SummarySchema>
```

Called only by `aiProvider.js` — never imported directly elsewhere.

**External API:** `POST https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key={key}`

## 10. UI / UX

This feature has no direct UI. Results are consumed by:
- `ResultsTable.jsx` — displays top-level fields in columns
- `CandidateDetail.jsx` — displays full schema with source quotes

The only user-visible output is the populated results table and the source quote tooltip on each field.

## 11. AI / ML Considerations

- **Model:** gemini-1.5-flash
- **Prompt location:** `src/main/parser/prompts.js` → `EXTRACTION_PROMPT`
- **Temperature:** 0.1 (near-deterministic)
- **Max tokens:** 4000
- **Fallback:** If Gemini fails after retry, surface error in UI and allow user to switch to Claude in Settings
- **PII handling:** Resume text is sent to Google. User must be informed in Settings panel.
- **Cost:** Free tier — 1,500 requests/day. No cost unless user exceeds this.
- **Eval metric:** All AC above must pass on a set of 10 real test resumes before shipping

## 12. Integration Points

- `src/main/parser/prompts.js` — prompt templates
- `src/main/parser/aiProvider.js` — calls `extractWithGemini`
- `src/main/db/database.js` — receives extracted JSON for storage
- Google Gemini API v1beta

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Gemini free tier rate limit hit during batch | M | M | Concurrency cap of 3 parallel AI calls; queue the rest |
| Gemini changes response format | L | H | Wrap all field access in optional chaining; log raw response on parse failure |
| Response truncated mid-JSON | M | H | maxOutputTokens already set to 4000; add JSON repair fallback |
| Google deprecates gemini-1.5-flash free tier | L | H | Claude fallback already built; just flip the default in Settings |
