## 1. Problem Statement

After resume extraction and public-source search complete, the recruiter needs a concise narrative: match confidence, outreach suggestions, discrepancies, and recommended follow-up queries — grounded only in data already collected.

## 2. Goals

- Add `SUMMARY_PROMPT` to `prompts.js` (shared by Gemini and Claude)
- Implement `summarizeWithGemini` / `summarizeWithClaude` and expose via `aiProvider.summarizeFindings(data)`
- Pass combined object: extracted resume fields + search results + LinkedIn + public records
- Persist summary JSON on the candidate record

## 3. Functional Requirements

- **FR-1.** Summary MUST return JSON: `summary`, `match_confidence`, `best_outreach_method`, `contact_hints`, `discrepancies`, `recommended_search_queries`.
- **FR-2.** Prompt MUST forbid inference beyond provided data.
- **FR-3.** UI MUST label which AI provider generated the summary.

## 4. Acceptance Criteria

- [ ] Full search pipeline triggers summarization automatically
- [ ] Detail drawer shows AI summary section with provider badge
- [ ] Sparse data yields explicit “insufficient data” in summary, not invented facts
