## 1. Problem Statement

The app currently requires Node.js, npm, and a terminal to run — completely inaccessible to a non-technical recruiter. It needs to be packaged into a standalone installer: a `.exe` for Windows and a `.dmg` for macOS, that installs and runs like any normal desktop app with no command-line interaction.

## 2. Goals

- Produce a Windows `.exe` installer
- Produce a macOS `.dmg` installer
- App launches directly from the desktop with no terminal
- Gemini API key is entered via the Settings panel on first launch — no environment variables
- Playwright Chromium is bundled inside the app

## 3. Non-Goals

- Code signing (can be added later for production)
- Auto-update mechanism
- Microsoft Store or Mac App Store distribution
- Linux packaging

## 4. Functional Requirements

- **FR-1.** The system MUST use `electron-builder` to package the app for Windows (NSIS installer) and macOS (DMG).
- **FR-2.** The packaged app MUST bundle Playwright's Chromium browser inside the app resources.
- **FR-3.** The packaged app MUST read the Gemini API key from `electron-store` — NOT from environment variables.
- **FR-4.** The packaged app MUST store the SQLite database in `app.getPath('userData')` — a writable location that survives app updates.
- **FR-5.** The `package.json` MUST include a `build` script: `electron-builder --win --mac`.
- **FR-6.** The Windows installer MUST create a desktop shortcut and Start Menu entry.
- **FR-7.** The macOS DMG MUST include a drag-to-Applications shortcut.
- **FR-8.** The first-launch experience MUST open directly to the Settings panel if no Gemini API key is set, prompting the user to add one before uploading resumes.

## 5. Acceptance Criteria

- **AC-1.** *Given* a Windows machine with no Node.js installed, *when* the .exe installer is run and the app is opened, *then* the app launches successfully.
- **AC-2.** *Given* the recruiter enters their Gemini API key in Settings on first launch, *when* they drop a resume, *then* it is parsed successfully.
- **AC-3.** *Given* the app is updated to a new version, *when* the new version is installed, *then* previously parsed candidates are still present in the results table.

## 6. Non-Functional Requirements

- **Installer size** — Target under 300MB for the Windows installer (Chromium is large).
- **Launch time** — App MUST be fully interactive within 5 seconds of launch on a mid-range laptop.
- **Security** — electron-builder MUST be configured with `nodeIntegration: false` and `contextIsolation: true` — these should already be set but must be verified before packaging.

## 7. electron-builder Configuration

Add to `package.json`:

```json
{
  "build": {
    "appId": "com.resumeintel.app",
    "productName": "Resume Intel",
    "directories": {
      "output": "dist"
    },
    "win": {
      "target": "nsis",
      "icon": "resources/icon.ico"
    },
    "mac": {
      "target": "dmg",
      "icon": "resources/icon.icns"
    },
    "extraResources": [
      {
        "from": "node_modules/playwright/.local-browsers",
        "to": "browsers"
      }
    ]
  }
}
```

## 8. First-Launch Flow

```
App opens
    ↓
Check electron-store for geminiApiKey
    ↓
Key missing → open Settings panel with banner:
"Welcome to Resume Intel. Add your free Gemini API key to get started."
    ↓
Key present → open Upload view directly
```

## 9. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Playwright Chromium path differs in packaged app vs dev | H | H | Use `app.isPackaged` to switch Chromium path at runtime |
| better-sqlite3 native binary needs to be rebuilt for packaged app | H | H | Add `electron-rebuild` to build script |
| Windows Defender flags the installer | M | M | Code signing resolves this — add as a follow-up task |
| Packaged app still reads GEMINI_API_KEY env var instead of store | M | H | Audit all env var references before packaging; replace with electron-store reads |

## 10. Build Instructions

```bash
# Install electron-builder
npm install --save-dev electron-builder electron-rebuild

# Rebuild native modules for Electron
npx electron-rebuild

# Package for both platforms
npx electron-builder --win --mac

# Output: dist/Resume Intel Setup.exe, dist/Resume Intel.dmg
```

## 11. Rollout Plan

- Ships as Milestone 8 (final milestone)
- Test on a clean Windows VM with no Node.js before shipping to the recruiter
- Deliver as a shared folder link or USB — no app store submission needed for internal use

## 12. Test Plan

- **Manual** — Install on a clean Windows 11 VM; verify all features work with no Node.js present
- **Manual** — Install on macOS; verify Playwright Chromium launches for DDG search
- **Manual** — Verify SQLite data persists through an app update
- **Manual** — Verify first-launch Settings prompt appears when no API key is set
