## 1. Problem Statement

Currently parsed results exist only in memory and are lost when the app closes. The recruiter needs results to persist between sessions so they can process resumes over multiple days and return to previous results without re-uploading.

## 2. Goals

- Persist all candidate records to a local SQLite database
- Load all records on app start so the table is populated immediately
- Support basic CRUD: create, read, delete (no update needed)
- Database file lives in the OS app data directory — no user configuration needed

## 3. Functional Requirements

- **FR-1.** The system MUST create the SQLite database file on first launch using `better-sqlite3`.
- **FR-2.** The database file MUST be stored in Electron's `app.getPath('userData')` directory.
- **FR-3.** The system MUST create the `candidates` table on startup if it does not exist.
- **FR-4.** Every parsed candidate MUST be inserted into the database immediately after extraction completes.
- **FR-5.** The system MUST load all candidate records from the database on app start and send them to the renderer via IPC.
- **FR-6.** The system MUST support deleting a candidate record by ID.
- **FR-7.** The "Clear All Data" button in Settings MUST delete all rows from the candidates table.
- **FR-8.** JSON fields (extracted_fields, search_results, linkedin_data, public_records, ai_summary) MUST be serialized to string on write and deserialized on read.

## 4. Acceptance Criteria

- **AC-1.** *Given* a resume is parsed, *when* the app is closed and reopened, *then* the candidate record appears in the results table.
- **AC-2.** *Given* a candidate record is deleted, *when* the table refreshes, *then* that row no longer appears.
- **AC-3.** *Given* "Clear All Data" is confirmed, *when* the app restarts, *then* the results table is empty.

## 5. Data Model

```sql
CREATE TABLE IF NOT EXISTS candidates (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  file_name TEXT NOT NULL,
  raw_text TEXT,
  extracted_fields JSON,
  search_results JSON,
  linkedin_data JSON,
  public_records JSON,
  ai_summary JSON,
  ai_provider TEXT,
  confidence_score TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6. API Surface

**File:** `src/main/db/database.js`

```typescript
initDatabase(): void
insertCandidate(candidate: CandidateRecord): number  // returns new row id
getAllCandidates(): CandidateRecord[]
getCandidateById(id: number): CandidateRecord | null
deleteCandidate(id: number): void
clearAllCandidates(): void
```

IPC channels:
- `db:get-all` → `getAllCandidates()`
- `db:delete` → `deleteCandidate(id)`

## 7. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Database file corruption | L | H | better-sqlite3 uses WAL mode by default — resistant to corruption |
| Large raw_text values slow down queries | M | L | Index only on id and created_at; raw_text is never searched |
