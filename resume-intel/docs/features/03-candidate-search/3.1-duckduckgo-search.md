## 1. Problem Statement

After extracting resume fields, the app needs to find the candidate's public online presence — primarily their LinkedIn profile URL — using only the extracted signals. DuckDuckGo is the primary search mechanism because it requires no API key, has no enforced rate limit, and can be scraped via Playwright.

## 2. Goals

- Build targeted search queries from extracted resume fields
- Scrape DuckDuckGo HTML results page using Playwright
- Return top 5 results with title, snippet, and URL
- Identify which results are LinkedIn profile URLs
- Operate completely free with no API key

## 3. Non-Goals

- Scraping the actual pages found in results (that's feature 4.1 for LinkedIn)
- Google search (that's feature 3.2, the paid fallback)
- Storing search results in the DB (handled by feature 9.1)

## 4. Personas & User Stories

- **As a recruiter**, I want the app to automatically search for the candidate online after parsing so that I don't have to manually construct search queries.
- **As a recruiter**, I want to see the LinkedIn profile URL for the candidate so that I can visit it directly.

## 5. Functional Requirements

- **FR-1.** The system MUST build a DuckDuckGo search query using: most recent employer (quoted), graduation year (if present), school name (if present), most notable license (if present), and `site:linkedin.com/in`.
- **FR-2.** The system MUST scrape `https://html.duckduckgo.com/html/?q={encodedQuery}` using a hidden Playwright Chromium browser.
- **FR-3.** The system MUST extract title, snippet, and URL from each result using CSS selectors `.result__title`, `.result__snippet`, `.result__url`.
- **FR-4.** The system MUST return the top 5 results.
- **FR-5.** The system MUST add a 2-3 second randomized delay between searches when processing a batch to avoid rate limiting.
- **FR-6.** The system MUST identify any result URL containing `linkedin.com/in/` as a LinkedIn profile candidate.
- **FR-7.** The system SHOULD run a second query without `site:linkedin.com/in` if the first query returns zero results, to find other public profiles.
- **FR-8.** The system MUST close the Playwright browser instance after each search to avoid resource leaks.

## 6. Non-Functional Requirements

- **Performance** — Each DDG search MUST complete within 10 seconds.
- **Reliability** — If DDG returns no results or times out, the feature MUST fail gracefully and allow the pipeline to continue to the next step.
- **Resource** — Playwright browser MUST be launched with `headless: true` and closed after every search.

## 7. Acceptance Criteria

- **AC-1.** *Given* a resume with employer "LBYD Federal" and school "Auburn University", *when* DDG search runs, *then* at least one result with a `linkedin.com/in/` URL is returned.
- **AC-2.** *Given* DDG returns zero results, *when* the search completes, *then* the pipeline continues without error and search_results is stored as an empty array.
- **AC-3.** *Given* a batch of 5 resumes, *when* searches run, *then* each search has at least a 2 second gap between it and the next.
- **AC-4.** *Given* a result URL containing `linkedin.com/in/`, *when* results are returned, *then* that result is flagged with `isLinkedIn: true`.

## 8. Data Model

Search results shape (stored in `search_results` JSON column):

```typescript
{
  query: string,
  results: Array<{
    title: string,
    snippet: string,
    url: string,
    isLinkedIn: boolean
  }>,
  searchedAt: string  // ISO timestamp
}
```

## 9. API Surface

```typescript
searchDuckDuckGo(extractedFields: ExtractionSchema): Promise<DDGSearchResult>
buildQuery(extractedFields: ExtractionSchema): string
```

Called by `ipc-handlers.js` as part of the `search:run` pipeline.

## 10. UI / UX

Shown in `SearchStatus.jsx` as a progress step: "Searching DuckDuckGo..."

Results shown in `CandidateDetail` drawer under "Search Results" section:
- Each result as a card: title, snippet, URL (clickable to open in system browser via `shell.openExternal`)
- LinkedIn results highlighted with LinkedIn logo icon

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| DuckDuckGo changes its HTML structure | M | H | Use multiple fallback selectors; log raw HTML on parse failure |
| Playwright conflicts with Electron's Chromium | M | H | Use Electron's built-in BrowserWindow as fallback if Playwright fails |
| DDG temporarily blocks IP during batch processing | M | M | Randomized delay between searches; exponential backoff on failure |
| No results found for candidate | H | L | Graceful empty state; pipeline continues to other search steps |
