## 1. Problem Statement

DuckDuckGo is free but sometimes returns zero results for sparse scrubbed resumes. Users with a Google Custom Search API key and Programmable Search Engine (CX) ID can opt in to a paid fallback that runs only when DDG finds nothing.

## 2. Goals

- Fall back to Google Custom Search JSON API when DDG returns zero results
- Reuse the same query builder as feature 3.1
- Return the same search result shape stored in `search_results`
- Skip Google entirely when API key or CX ID is not configured

## 3. Non-Goals

- Replacing DuckDuckGo as the primary search path
- Scraping Google HTML results pages
- Requiring Google keys for normal operation

## 4. Personas & User Stories

- **As a recruiter**, I want Google search to run automatically when DDG finds nothing so that I still have a chance to locate a LinkedIn profile.
- **As a recruiter**, I want Google search to stay optional so that I am not forced to pay for an API key.

## 5. Functional Requirements

- **FR-1.** The system MUST call Google Custom Search JSON API via raw `fetch()` when DDG returns zero results.
- **FR-2.** The system MUST read `googleSearchApiKey` and `googleCxId` from `electron-store`.
- **FR-3.** The system MUST skip Google search when either key is missing or blank.
- **FR-4.** The system MUST reuse `buildQuery()` from `duckduckgo.js` for the Google query (LinkedIn site filter included).
- **FR-5.** The system MUST map API items to `{ title, snippet, url, isLinkedIn }` and return at most 5 results.
- **FR-6.** The system MUST set `source: "google"` on the search result object when Google supplied the hits.
- **FR-7.** The system MUST fail gracefully (empty results + optional error message) if the Google API returns an error.
- **FR-8.** The system MUST NOT call Google when DDG already returned one or more results.

## 6. Non-Functional Requirements

- **Performance** — Google API call MUST complete within 10 seconds.
- **Cost** — Google is only invoked on DDG miss; never on every upload.
- **Privacy** — Query text is sent to Google only when fallback runs and keys are configured.

## 7. Acceptance Criteria

- **AC-1.** *Given* DDG returns zero results and Google API key + CX ID are configured, *when* search runs, *then* stored `search_results.source` is `"google"` and at least one result is returned.
- **AC-2.** *Given* DDG returns zero results and Google keys are not configured, *when* search runs, *then* `search_results.results` is an empty array and no error is thrown.
- **AC-3.** *Given* Google returns a LinkedIn profile URL, *when* results are stored, *then* that item has `isLinkedIn: true`.
- **AC-4.** *Given* DDG returns at least one result, *when* search runs, *then* Google is not called and `source` is omitted or `"duckduckgo"`.

## 8. Data Model

Same as 3.1, with optional `source`:

```typescript
{
  query: string,
  results: Array<{ title, snippet, url, isLinkedIn }>,
  searchedAt: string,
  source?: "duckduckgo" | "google",
  error?: string
}
```

## 9. API Surface

**File:** `src/main/search/googleSearch.js`

```typescript
searchGoogle(extractedFields: object): Promise<SearchResult | null>
parseGoogleItems(items: unknown): ResultItem[]
hasGoogleSearchConfig(): boolean
```

Called from `ipc-handlers.js` inside `runSearchForCandidate` after DDG.

**External API:** `GET https://www.googleapis.com/customsearch/v1?key={key}&cx={cx}&q={query}&num=5`

## 10. UI / UX

No new settings fields (already in 8.1). Candidate detail drawer MAY show `data-testid="search-source"` when `source` is present.

## 11. Integration Points

- `src/main/search/duckduckgo.js` — `buildQuery`
- `src/main/ipc-handlers.js` — search pipeline
- Settings keys: `googleSearchApiKey`, `googleCxId`
