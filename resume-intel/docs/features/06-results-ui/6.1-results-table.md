## 1. Problem Statement

The recruiter needs to see all parsed resumes in a scannable table that surfaces the most important signals at a glance — without opening every individual record. The current table shows only a handful of fields and lacks sorting, filtering, and row actions.

## 2. Goals

- Display all parsed candidates in a sortable, filterable table
- Show the most useful fields as columns without overwhelming the recruiter
- Provide per-row actions: view detail, re-search, delete, export row
- Color-code by confidence score at a glance

## 3. Functional Requirements

- **FR-1.** The table MUST display columns: File Name, Summary Title, Most Recent Employer, Most Recent Title, Location, Education, Licenses, Years Experience, LinkedIn Found (✓/✗), Confidence Score, Actions.
- **FR-2.** Every column MUST be sortable (click header to sort asc/desc).
- **FR-3.** The recruiter MUST be able to filter by: confidence score, LinkedIn found, license type.
- **FR-4.** Each row MUST have action buttons: "View Detail", "Re-run Search", "Delete", "Export Row".
- **FR-5.** Confidence score MUST be color coded: green (high), amber (medium), red (low).
- **FR-6.** The table MUST persist sort and filter state during the session.
- **FR-7.** The table MUST load data from the SQLite database on app start, not only from in-memory state.

## 4. Acceptance Criteria

- **AC-1.** *Given* 10 parsed resumes, *when* the table loads, *then* all 10 rows appear with all required columns populated.
- **AC-2.** *Given* the recruiter clicks the "Confidence" column header, *when* sorted, *then* rows reorder by high → medium → low.
- **AC-3.** *Given* the recruiter filters by "LinkedIn Found: Yes", *when* filter is applied, *then* only rows with a LinkedIn URL appear.
- **AC-4.** *Given* a row with confidence "high", *when* rendered, *then* the confidence badge is green.

## 5. UI / UX

- Dark sidebar (navigation/filters), light main content area
- Sticky header row
- Hover state on rows
- Row click opens CandidateDetail drawer (feature 6.2)
- Empty state: "No resumes yet — drop files above to get started"
